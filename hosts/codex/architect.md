# Codex Architect

This file explains how the architect phase should behave when Codex is the host orchestrator.

## Shared basis

Read first:
- `skills/_craft-core/host-policy.md`
- `skills/_craft-core/project-context.md`
- `skills/_craft-core/develop-routing.md`

## Expectations

- Codex owns planning and requirement clarification in this mode.
- Plans should be written with a strict Codex quality bar:
  - exact file paths
  - clear task boundaries
  - explicit sequencing and dependencies
  - reuse-first decisions

## Tier guidance

- architectural planning and plan review are usually `premium`
- bounded context gathering is usually `balanced`
