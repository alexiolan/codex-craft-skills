# Codex Brainstorm

This file explains how the brainstorm phase should behave when Codex is the host orchestrator.

## Shared basis

Read first:
- `skills/_craft-core/runtime-model.md`
- `skills/_craft-core/host-policy.md`
- `skills/_craft-core/project-context.md`

## Expectations

- Codex leads the brainstorm conversation directly with the user.
- The same workflow commitments apply as on other hosts:
  - explore targeted context first
  - ask clarifying questions when ambiguity matters
  - propose multiple approaches with tradeoffs
  - do not start implementation before design approval

## Tier guidance

- Brainstorming and design exploration are usually `premium`.
- Supporting codebase exploration is usually `balanced`.
- Narrow follow-up checks can use `fast` or `balanced` depending on ambiguity.
