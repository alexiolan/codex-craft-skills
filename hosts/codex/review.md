# Codex Review

This file explains how review loops should behave when Codex is the host orchestrator.

## Shared basis

Read first:
- `skills/_craft-core/host-policy.md`
- `skills/_craft-core/project-context.md`

## Expectations

- Review still happens before implementation approval and after implementation work.
- The current host is responsible for final judgment even when local-model assistance is enabled.
- Review output should stay specific, evidence-based, and scoped to actual risks or inconsistencies.

## Tier guidance

- spec review and plan review are usually `premium`
- post-implementation review is usually `balanced`
- mechanical verification follow-up can use `fast`
